📚 Quick Reference Guide

🆔 ID Attribute

  • Syntax: id="unique-name"
  • CSS Selector: #unique-name { }
  • Rule: Only ONE element per page can have a specific ID
  • Use for: Unique elements like headers, main content, footer

📦 Class Attribute

  • Syntax: class="group-name"
  • CSS Selector: .group-name { }
  • Multiple classes: class="class1 class2 class3"
  • Use for: Reusable styles, component groups, shared styling
Feature ID (#) Class (.)
Uniqueness Must be unique on page Can be reused many times
Per Element Only 1 ID allowed Multiple classes allowed
CSS Priority Higher (100 points) Lower (10 points)
Example <div id="header"> <div class="card">

Task 1: Basic ID Usage 15 Points

Create a page with unique sections using ID attributes.

Requirements:

  • Create a <header> with id="site-header"
  • Create a <main> with id="main-content"
  • Create a <footer> with id="site-footer"
  • Style each section with different background colors

Example Code:

<style>
    #site-header {
        background-color: #667eea;
        color: white;
        padding: 20px;
        text-align: center;
    }
    
    #main-content {
        padding: 40px;
        min-height: 300px;
    }
    
    #site-footer {
        background-color: #333;
        color: white;
        padding: 20px;
        text-align: center;
    }
</style>

<header id="site-header">
    <h1>My Website</h1>
</header>

<main id="main-content">
    <p>Welcome to my website content...</p>
</main>

<footer id="site-footer">
    <p>© 2024 My Website</p>
</footer>

💡 Pro Tips:

  • Use meaningful names: nav-link is better than link1
  • Keep ID names unique and descriptive
  • Use hyphens for multi-word names: main-content
  • Classes are for styling groups, IDs are for unique elements
  • Test your page in the browser as you code!
  • Use browser DevTools (F12) to inspect your elements
