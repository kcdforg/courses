♿ Lesson 15: Web Accessibility

Learn to make your websites accessible to everyone, including people with disabilities. Good accessibility makes the web better for ALL users!

What is Web Accessibility?

Web accessibility (often abbreviated as A11y) means making websites usable by everyone, including people with visual, auditory, motor, or cognitive disabilities. It's not just the right thing to do—it's often legally required and benefits all users.

🎯 Why Accessibility Matters

  • 15% of the world has some form of disability
  • Legal requirement in many countries (ADA, Section 508)
  • Better UX for everyone (clear labels, good contrast)
  • Improved SEO (semantic HTML, alt text)
  • Wider audience reach for your content

Key Accessibility Principles:

1. Alt Text: Describe images for screen readers

2. Labels: Associate labels with form inputs

3. Semantic HTML: Use meaningful tags

4. Keyboard Navigation: Tab through all interactive elements

5. Color Contrast: Sufficient contrast for readability

🎯 Progressive Activities

Complete these activities to make your websites accessible!

Activity 1

Add Alt Text to Images

Beginner

Learn to add descriptive alt text to images for screen reader users.

Instructions:

  1. Add images to your page
  2. Include descriptive alt attribute for each
  3. Describe what's IN the image, not just the filename
  4. Keep alt text concise but descriptive
<html lang="en">
<body>

<h1>My Pet Gallery</h1>

<!-- Good alt text examples -->
<img src="golden-retriever.jpg" 
     alt="Golden retriever playing with a ball in the park">

<img src="product.jpg" 
     alt="Red laptop computer with 15-inch screen">

<!-- Decorative image - empty alt -->
<img src="border-decoration.png" alt="">

</body>
</html>

Expected Output:

Screen readers will announce the alt text, helping visually impaired users understand your images.

💡 Hints:

  • Be descriptive but concise
  • Don't say "image of" or "picture of"
  • Use alt="" for purely decorative images
  • Include text if image contains important text
Activity 2

Use Proper Labels for Forms

Beginner

Always associate labels with form inputs for better accessibility.

Instructions:

  1. Create a form with multiple inputs
  2. Add <label> for EACH input
  3. Connect label and input using for and id
  4. Test: clicking label should focus the input
<html lang="en">
<body>

<h2>Contact Form</h2>

<form>
    <label for="name">Full Name:</label><br>
    <input type="text" id="name" name="name"><br><br>
    
    <label for="email">Email Address:</label><br>
    <input type="email" id="email" name="email"><br><br>
    
    <label for="message">Your Message:</label><br>
    <textarea id="message" name="message" rows="4"></textarea><br><br>
    
    <button type="submit">Send Message</button>
</form>

</body>
</html>

Expected Output:

Screen readers announce the label when users focus on inputs. Clicking the label focuses the input.

💡 Hints:

  • The for attribute must match the input's id
  • Labels make forms easier to use for everyone
  • Never use placeholder as a replacement for labels
Activity 3

Use Semantic HTML

Intermediate

Replace generic divs with meaningful semantic HTML tags.

Instructions:

  1. Create a page structure using semantic tags
  2. Use <header>, <nav>, <main>, <footer>
  3. Use <article> for blog posts or news items
  4. Add lang="en" to html tag
<html lang="en">
<body>

<header>
    <h1>My Blog</h1>
    <nav>
        <ul>
            <li><a href="index.html">Home</a></li>
            <li><a href="about.html">About</a></li>
            <li><a href="contact.html">Contact</a></li>
        </ul>
    </nav>
</header>

<main>
    <article>
        <h2>My First Blog Post</h2>
        <p>This is the content of my blog post.</p>
    </article>
    
    <article>
        <h2>Another Blog Post</h2>
        <p>More interesting content here.</p>
    </article>
</main>

<footer>
    <p>© 2025 My Blog. All rights reserved.</p>
</footer>

</body>
</html>

Expected Output:

Screen readers can navigate by landmarks (header, nav, main, footer), making the site much easier to use.

💡 Hints:

  • Semantic tags help screen readers understand page structure
  • Users can jump directly to navigation or main content
  • Always include lang attribute on html tag
Activity 4

Write Descriptive Link Text

Intermediate

Make your links accessible by using descriptive link text instead of "click here".

Instructions:

  1. Create links with descriptive text
  2. AVOID generic phrases like "click here" or "read more"
  3. Link text should make sense out of context
  4. Include the destination or purpose in the link text
<html lang="en">
<body>

<h2>Recommended Resources</h2>

<!-- BAD: Generic link text -->
<p>To learn more about HTML, <a href="tutorial.html">click here</a>.</p>

<!-- GOOD: Descriptive link text -->
<p>Check out our <a href="tutorial.html">complete HTML tutorial</a>.</p>

<!-- BAD: "Read more" without context -->
<p>We offer free courses. <a href="courses.html">Read more</a></p>

<!-- GOOD: Descriptive and contextual -->
<p><a href="courses.html">Browse our free web development courses</a></p>

<!-- GOOD: Link text describes destination -->
<p>Download the <a href="guide.pdf">HTML quick reference guide (PDF)</a></p>

</body>
</html>

Expected Output:

Links that make sense when read out of context by screen reader users.

💡 Hints:

  • Screen reader users often navigate by links only
  • Link text should describe where the link goes
  • Avoid "click here", "read more", "more info"
  • Indicate file type/size if linking to documents
Activity 5

Build a Fully Accessible Page

Advanced

Create a complete webpage that follows all accessibility best practices.

Instructions:

  1. Use semantic HTML throughout
  2. All images have alt text
  3. All forms have proper labels
  4. Links are descriptive
  5. Proper heading hierarchy (h1→h2→h3)
  6. Include lang attribute
  7. Test keyboard navigation (Tab key)
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Accessible Website Example</title>
</head>
<body>

<header>
    <h1>Accessible Web Design</h1>
    <nav>
        <ul>
            <li><a href="#about">About</a></li>
            <li><a href="#services">Services</a></li>
            <li><a href="#contact">Contact</a></li>
        </ul>
    </nav>
</header>

<main>
    <section id="about">
        <h2>About Us</h2>
        <img src="team.jpg" 
             alt="Our team of five developers working together at a modern office">
        <p>We create websites that everyone can use.</p>
    </section>

    <section id="services">
        <h2>Our Services</h2>
        <article>
            <h3>Web Accessibility Audits</h3>
            <p>We review your site for accessibility issues.</p>
            <a href="services/audits.html">Learn about accessibility audits</a>
        </article>
    </section>

    <section id="contact">
        <h2>Contact Us</h2>
        <form>
            <label for="fullname">Full Name:</label><br>
            <input type="text" id="fullname" name="fullname" required><br><br>
            
            <label for="email">Email Address:</label><br>
            <input type="email" id="email" name="email" required><br><br>
            
            <button type="submit">Send Message</button>
        </form>
    </section>
</main>

<footer>
    <p>© 2025 Accessible Web Design. All rights reserved.</p>
</footer>

</body>
</html>

Expected Output:

A fully accessible page that works perfectly with screen readers, keyboard navigation, and assistive technologies.

🏆 Challenge:

  • Test with a screen reader (NVDA on Windows, VoiceOver on Mac)
  • Navigate using only keyboard (Tab, Enter, Space)
  • Add ARIA labels where needed
  • Check color contrast using a contrast checker tool

📚 Accessibility Checklist

✓ Essential Checks:

  • ✅ All images have meaningful alt text
  • ✅ All form inputs have associated labels
  • ✅ Proper heading hierarchy (H1→H2→H3)
  • ✅ Sufficient color contrast (4.5:1 for normal text)
  • ✅ Keyboard navigation works everywhere
  • ✅ Links have descriptive text (no "click here")
  • ✅ Semantic HTML tags used
  • ✅ Language declared (lang="en")
  • ✅ Videos have captions/transcripts
