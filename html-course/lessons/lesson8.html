📝 Lesson 8: HTML Lists

Learn to organize information using ordered, unordered, and description lists

📚 Understanding HTML List Tags

HTML provides three types of lists to organize and display information in a structured way. Lists are essential for navigation menus, step-by-step instructions, feature lists, and much more.

🎯 Key Points:
  • <ul> - Unordered List (bullet points)
  • <ol> - Ordered List (numbered or lettered)
  • <dl> - Description List (terms and definitions)
  • <li> - List Item (used in ul and ol)
  • Lists can be nested inside each other
  • Lists can be styled with CSS for custom appearances

Unordered List

<ul>
  <li>Item</li>
  <li>Item</li>
</ul>

Output:

  • Item
  • Item

Ordered List

<ol>
  <li>First</li>
  <li>Second</li>
</ol>

Output:

  1. First
  2. Second

Description List

<dl>
  <dt>Term</dt>
  <dd>Definition</dd>
</dl>

Output:

Term
Definition

Ordered List Types:

Type Description Example
type="1" Numbers (default) 1, 2, 3, 4...
type="A" Uppercase letters A, B, C, D...
type="a" Lowercase letters a, b, c, d...
type="I" Uppercase Roman numerals I, II, III, IV...
type="i" Lowercase Roman numerals i, ii, iii, iv...

🎯 Hands-On Activities

1

Create Your Shopping List Beginner

Goal: Create an unordered list of grocery items.

📝 Instructions:

  1. Create a new HTML file called shopping-list.html
  2. Add a heading "My Shopping List"
  3. Create an unordered list with at least 8 grocery items
  4. Group related items together (fruits, vegetables, dairy, etc.)
  5. Open in browser to see your list with bullet points

✅ Expected Output:

A bulleted list of grocery items displayed on your page.

  • Apples
  • Bananas
  • Milk
  • Bread
💡 Need a hint?
<h2>My Shopping List</h2>
<ul>
<li>Apples</li>
<li>Bananas</li>
<li>Milk</li>
</ul>
2

Create Step-by-Step Instructions Beginner

Goal: Create an ordered list with sequential steps.

📝 Instructions:

  1. Create a file called recipe.html
  2. Choose a simple recipe (sandwich, smoothie, instant noodles)
  3. Add a heading with the recipe name
  4. Create an unordered list for ingredients
  5. Create an ordered list for step-by-step instructions
  6. Make sure steps are in logical order
<h2>How to Make a Peanut Butter Sandwich</h2>
<h3>Ingredients:</h3>
<ul>
<li>2 slices of bread</li>
<li>Peanut butter</li>
<li>Jelly (optional)</li>
</ul>
<h3>Instructions:</h3>
<ol>
<li>Take two slices of bread</li>
<li>Spread peanut butter on one slice</li>
<li>Spread jelly on the other slice</li>
<li>Press slices together</li>
<li>Enjoy your sandwich!</li>
</ol>

✅ Expected Output:

A recipe page with a bulleted ingredient list and numbered step-by-step instructions.

🚀 Challenge:

Add a "Tips" section with another unordered list of helpful cooking tips!

3

Experiment with List Types Intermediate

Goal: Create ordered lists with different numbering styles.

📝 Instructions:

  1. Create a file called list-types.html
  2. Add a heading "Different List Styles"
  3. Create 5 separate ordered lists, each with a different type attribute:
    • Numbers (type="1")
    • Uppercase letters (type="A")
    • Lowercase letters (type="a")
    • Uppercase Roman numerals (type="I")
    • Lowercase Roman numerals (type="i")
  4. Each list should have at least 5 items
  5. Add descriptive headings before each list explaining the type
<h3>Alphabetical List (Uppercase)</h3>
<ol type="A">
<li>First item</li>
<li>Second item</li>
<li>Third item</li>
</ol>
<h3>Roman Numerals (Lowercase)</h3>
<ol type="i">
<li>First item</li>
<li>Second item</li>
<li>Third item</li>
</ol>

✅ Expected Output:

Five lists with different numbering styles: 1,2,3... A,B,C... a,b,c... I,II,III... i,ii,iii...

🚀 Challenge:

Use the start attribute to make a list begin at number 5 or letter E!

4

Create Nested Lists Intermediate

Goal: Create multi-level nested lists to show hierarchy.

📝 Instructions:

  1. Create a file called school-subjects.html
  2. Add a heading "My School Subjects"
  3. Create an unordered list of 3-4 main subject categories (Science, Math, Arts, etc.)
  4. For each subject, add a nested unordered list with specific topics or subtopics
  5. Make sure to properly indent your code for readability
  6. Verify that the nested lists display with different bullet styles
<h2>My School Subjects</h2>
<ul>
<li>Science
<ul>
<li>Biology</li>
<li>Chemistry</li>
<li>Physics</li>
</ul>
</li>
<li>Mathematics
<ul>
<li>Algebra</li>
<li>Geometry</li>
<li>Calculus</li>
</ul>
</li>
</ul>

✅ Expected Output:

A hierarchical list where main subjects have solid bullets and sub-subjects have hollow circles or other bullet styles.

🚀 Challenge:

Create a three-level nested list: Subjects → Topics → Subtopics (e.g., Science → Biology → Cell Biology)

5

Build a Description List (Glossary) Intermediate

Goal: Create a glossary using description lists.

📝 Instructions:

  1. Create a file called html-glossary.html
  2. Add a heading "HTML Terms Glossary"
  3. Create a description list (<dl>) with at least 8 HTML terms
  4. For each term:
    • Use <dt> for the term name
    • Use <dd> for the definition
  5. Include terms like: HTML, CSS, Tag, Attribute, Element, Browser, etc.
  6. Write clear, beginner-friendly definitions
<h2>HTML Terms Glossary</h2>
<dl>
<dt>HTML</dt>
<dd>HyperText Markup Language - the standard language for creating web pages</dd>
<dt>CSS</dt>
<dd>Cascading Style Sheets - used to style and layout web pages</dd>
<dt>Tag</dt>
<dd>An HTML element wrapped in angle brackets, like &lt;p&gt; or &lt;div&gt;</dd>
<dt>Attribute</dt>
<dd>Provides additional information about an element, like href or src</dd>
</dl>

✅ Expected Output:

A formatted glossary where terms appear in bold (or styled) and definitions are indented below each term.

🚀 Challenge:

Style your glossary with CSS: make terms bold and colored, add spacing between entries, and add a background color to definitions!

6

Mix Lists: Create a Course Outline Advanced

Goal: Combine ordered and unordered lists to create a detailed course outline.

📝 Instructions:

  1. Create a file called course-outline.html
  2. Create a course outline for "Introduction to Web Development"
  3. Use an ordered list for the main modules/weeks (numbered)
  4. For each module, add a nested unordered list of topics
  5. For some topics, add another nested list of subtopics or learning objectives
  6. Include at least 4 modules with 3-5 topics each
  7. Add appropriate headings and styling
<h2>Introduction to Web Development - Course Outline</h2>
<ol>
<li><strong>Week 1: HTML Basics</strong>
<ul>
<li>Introduction to HTML
<ul>
<li>What is HTML?</li>
<li>HTML Structure</li>
</ul>
</li>
<li>Basic Tags</li>
<li>Attributes</li>
</ul>
</li>
<li><strong>Week 2: CSS Fundamentals</strong>
<ul>
<li>CSS Syntax</li>
<li>Selectors</li>
<li>Colors and Fonts</li>
</ul>
</li>
</ol>

✅ Expected Output:

A well-structured course outline with numbered modules and bulleted topics/subtopics showing clear hierarchy.

🚀 Challenge:

Add a summary section using a description list that defines key terms for each module!

7

Final Project: Personal Resume Page Advanced

Goal: Create a complete resume page using all three types of lists.

📝 Project Requirements:

  1. Personal Information: Name, contact details, professional summary
  2. Skills Section: Use an unordered list organized by categories (Technical Skills, Soft Skills, Languages)
  3. Work Experience: Use an ordered list (most recent first) with:
    • Job title and company
    • Nested unordered list of responsibilities
  4. Education: Use an ordered list (most recent first) with nested lists for achievements or coursework
  5. Projects: Use an unordered list with nested lists describing each project
  6. Certifications: Use a description list with certificate names and issuing organizations
  7. Styling: Add CSS to make it look professional:
    • Change list bullet styles
    • Add spacing and margins
    • Use different colors for sections
    • Consider using custom list-style-type or images as bullets

✅ Project Checklist:

  • ☐ All three list types are used appropriately
  • ☐ Lists are properly nested where needed
  • ☐ Information is well-organized and easy to read
  • ☐ CSS styling makes the resume professional
  • ☐ No spelling or grammar errors
  • ☐ HTML is properly indented
  • ☐ All tags are correctly closed
  • ☐ Page displays correctly in browser
💡 Styling Tips

CSS examples you can use:

/* Remove bullets from main list */
ul.no-bullets {
list-style-type: none;
}
/* Square bullets */
ul.square {
list-style-type: square;
}
/* Add spacing */
li {
margin-bottom: 15px;
}

  • Make the resume printable with a print stylesheet
  • Add a timeline visualization using lists and CSS
  • Create a "References" section with description lists
